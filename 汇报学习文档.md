## 一、项目概述：Semantic Insight 是什么？

### 1.1 一句话概括

**Semantic Insight（语义洞察）是一个基于深度学习的双任务中文自然语言处理（NLP）系统，能同时完成"文本分类"和"命名实体识别（NER）"两个任务。**

### 1.2 它能做什么？

假设你输入一句话：

> **"今天NBA总决赛，洛杉矶湖人队大胜波士顿凯尔特人"**

系统会输出两个结果：

| 任务 | 输出 | 含义 |
|------|------|------|
| 📂 **文本分类** | `体育`（置信度 99.46%） | 判断这段话属于哪个类别 |
| 🏷 **命名实体识别** | `洛杉矶湖人队` → ORG（机构）<br>`波士顿凯尔特人` → ORG（机构） | 识别出文本中的人名、地名、机构名 |

### 1.3 应用场景

- **新闻自动分类**：自动将新闻分到体育、财经、科技等类别
- **信息抽取**：从文本中自动提取关键实体（人名、地名、机构名）
- **舆情分析**：监控社交媒体上的文本内容并分类
- **搜索引擎优化**：为文章自动打标签

### 1.4 技术栈

| 层级 | 使用技术 | 作用 |
|------|----------|------|
| 深度学习框架 | PyTorch 2.0 | 构建和训练神经网络 |
| 预训练模型 | bert-base-chinese（HuggingFace） | 提供强大的中文语义理解能力 |
| 前端 | FastAPI + 纯 HTML/CSS/JS | 提供 Web 交互界面 |
| 可视化 | Matplotlib + Seaborn | 绘制混淆矩阵、训练曲线等图表 |

> **一句话总结**：这是一个用 BERT 做中文文本"分类"和"实体识别"的 AI 系统，带 Web 界面。

## 二、核心概念扫盲

如果你是零基础，先理解以下概念：

### 2.1 什么是 NLP（自然语言处理）？

NLP 就是让计算机"理解"人类语言的技术。人类读"我很开心"知道这是正面情绪，但计算机只认识 0 和 1，NLP 就是在这之间架起的桥梁。

### 2.2 什么是 BERT？

**BERT**（Bidirectional Encoder Representations from Transformers）是 Google 在 2018 年发布的预训练语言模型。

- **通俗理解**：BERT 就像一个已经读过海量中文文本的"超级学霸"，它对中文词汇、语法、语义有很深的理解
- **预训练**：Google 已经帮我们用几百万本书和维基百科文章训练好了，我们只需要在自己的任务上"微调"即可
- **bert-base-chinese**：BERT 的中文版本，12 层 Transformer，隐藏层大小 768，总参数约 1.1 亿

打个比方：BERT 就像一个已经读完大学中文系的学生，你只需要花很少的时间教它做"分类"这个具体工作，它就能做得很好。

### 2.3 什么是文本分类？

给定一段文本，判断它属于哪个类别。

- 输入：`"湖人队获得了NBA总冠军"`
- 输出：`体育`（从10个类别中选出最可能的一个）

本项目支持 10 个类别：体育、财经、科技、教育、时尚、社会、游戏、房产、娱乐、时政。

### 2.4 什么是命名实体识别（NER）？

从文本中找出"有特殊含义的词语"，并判断它们的类型。

| 实体类型 | 含义 | 示例 |
|----------|------|------|
| PER | 人名 | 姚明、刘德华 |
| LOC | 地名 | 北京、深圳 |
| ORG | 机构名 | 华为、北京大学 |

### 2.5 什么是 BIO 标注？

NER 使用 BIO 标注体系来表示每个字的位置信息：

| 标签 | 含义 | 示例 |
|------|------|------|
| B-PER | 人名的开头（Begin） | 姚(B-PER) |
| I-PER | 人名的内部（Inside） | 明(I-PER) |
| B-LOC | 地名的开头 | 北(B-LOC) |
| I-LOC | 地名的内部 | 京(I-LOC) |
| B-ORG | 机构名的开头 | 华(B-ORG) |
| I-ORG | 机构名的内部 | 为(I-ORG) |
| O | 非实体（Outside） | 的、了、在 |

例如"姚明在上海打球"：
- 姚 → B-PER，明 → I-PER，在 → O，上 → B-LOC，海 → I-LOC，打 → O，球 → O

> **一句话总结**：BERT 提供理解能力，分类判断"这是什么类别"，NER 找出"这里有哪些实体"。

### 3.3 训练 vs 推理双 Encoder 架构

```
训练阶段：                        推理阶段（双 Encoder）：
┌──────────────┐                 ┌──────────────┐   ┌──────────────┐
│ Shared BERT  │                 │  CLS Encoder │   │  NER Encoder │
│   Encoder    │                 │ (分类训练权重)│   │ (NER训练权重) │
├──────┬───────┤                 ├──────────────┤   ├──────────────┤
│ CLS  │  NER  │                 │  Classifier  │   │  NER Linear  │
│ Head │  Head │                 │    Head      │   │     Head     │
└──────┴───────┘                 └──────────────┘   └──────────────┘
```

> **为什么推理时用两个 Encoder？** 因为分类和 NER 是分别训练的，各自有最优的 BERT 权重。如果共用一个 Encoder，加载分类权重就会覆盖 NER 权重（反之亦然），导致一个任务效果变差。双 Encoder 确保两个任务都能用各自最好的权重。

## 四、数据层详解

### 4.1 分类数据：THUCNews

**数据来源**：THUCNews 是清华大学推出的中文新闻分类数据集。

- 包含 10 个类别，每个类别约 2 万条新闻标题
- 总计约 20 万条数据
- 数据划分：80% 训练集 / 10% 验证集 / 10% 测试集

**处理流程**（`prepare_real_data.py`）：
1. 读取每个类别的 `.jsonl` 文件
2. 提取 `title` 字段作为文本
3. 过滤掉长度小于 4 的标题
4. 随机打乱并按 8:1:1 划分
5. 保存为 JSON 格式：`{"text": "新闻标题", "label": 类别编号}`

### 4.2 NER 数据：chinese_common_ner

**数据来源**：中文通用 NER 数据集，来自 HuggingFace。

- 3 种实体类型：PER（153条）、LOC（187条）、ORG（227条）
- 使用 BIO 标注体系，共 7 个标签（O + 3×B/I）

**NER 数据特殊处理**（`dataset.py` 的 `NERDataset`）：

由于 BERT 的中文分词器可能把一个词拆成多个子词（subword），需要进行标签对齐：

```
原始: ["姚明", "在", "上海"] → 标签: [B-PER, O, B-LOC]
BERT分词后: ["姚", "明", "在", "上", "海"] → 标签: [B-PER, I-PER, O, B-LOC, I-LOC]
```

每个子词都继承原词的标签，确保不丢失信息。

### 4.3 数据加载器设计

`dataset.py` 中的 `create_dataloaders()` 函数一次性创建 6 个 DataLoader：

| DataLoader | 用途 |
|------------|------|
| `cls_train` / `cls_val` / `cls_test` | 分类任务的训练/验证/测试 |
| `ner_train` / `ner_val` / `ner_test` | NER 任务的训练/验证/测试 |

> **一句话总结**：分类数据来自清华大学 THUCNews 新闻集（20万条），NER 数据来自开源中文 NER 数据集，经过统一预处理后送入 BERT 模型。

## 五、模型层详解

### 5.1 共享编码器（`encoder.py`）

```python
class SharedEncoder(nn.Module):
    def __init__(self, model_name="bert-base-chinese", 
                 freeze_embeddings=True, freeze_layers=6):
```

- 加载预训练的 `bert-base-chinese`
- 支持**冻结（freeze）**策略：冻结 embedding 层和前 N 层 Transformer
- 冻结 = 不更新这些参数，只训练后面的层，节省计算资源
- 最终配置：`freeze_embeddings=false, freeze_layers=0`（全部解冻，最佳效果）

**输出**：
- `last_hidden_state`：(batch, seq_len, 768) — 每个 token 的向量表示（NER 用）
- `pooler_output`：(batch, 768) — [CLS] token 的向量（分类用）

### 5.2 分类头（`classifier.py`）

```
输入文本 → BERT Encoder → [CLS]向量(768维) → Dropout → Linear(768→10) → 10个类别的分数
```

- 取 BERT 输出的 `pooler_output`（即 [CLS] token 的表示）
- 经过一个 Dropout（防止过拟合）
- 经过一个线性层将 768 维映射到 10 维
- 用 CrossEntropyLoss（交叉熵损失）计算损失

### 5.3 NER 标注头（`ner_tagger.py`）

```
输入文本 → BERT Encoder → 每个Token的向量(768维) → Dropout → Linear(768→7) → 7个BIO标签的分数
```

- 使用 BERT 的 `last_hidden_state`（每个 token 都有 768 维向量）
- 对每个 token 预测一个 BIO 标签
- 支持两种损失函数：
  - **CE（CrossEntropyLoss）**：本项目实际使用，配合类别权重处理不平衡
  - **CRF（Conditional Random Field）**：条件随机场，能约束标签转移（如 I-PER 不能接在 O 后面直接出现）

### 5.4 多任务推理模型（`multitask_model.py`）

- 整合分类头和 NER 头，提供统一的 `predict(text)` 接口
- BIO 标签解析：将 `[B-ORG, I-ORG, O, B-LOC, I-LOC]` 转为 `[{word:"华为", type:"ORG"}, {word:"深圳", type:"LOC"}]`
- 完整流程：
  1. Tokenize（分词）→ 2. BERT 编码 → 3. 分类预测（取最大概率）→ 4. NER 标签预测 → 5. BIO 解析 → 6. 返回结果

### 5.5 双 Encoder 推理模型（`server.py` 的 `DualEncoderModel`）

- 分类和 NER 各自加载独立的 BERT encoder
- 分类 encoder 加载 `best_classification.pt` 权重
- NER encoder 加载 `best_ner.pt` 权重
- 互不干扰，保证两个任务都用最优权重

## 六、训练流程详解

### 6.1 通用训练器（`trainer.py`）

这是整个训练的核心，负责：

| 组件 | 作用 |
|------|------|
| **AdamW 优化器** | 调整模型参数，使损失函数最小化 |
| **线性学习率调度** | 先预热（warmup）再线性衰减，让训练更稳定 |
| **梯度累积** | 累积 2 步再更新参数，等效于增大 batch size |
| **梯度裁剪** | 防止梯度爆炸，限制最大梯度范数为 1.0 |
| **定期评估** | 每 200 步在验证集上评估，保存最佳模型 |
| **训练历史记录** | 记录 loss 和 metric 变化曲线 |

### 6.2 分类训练（`train_classifier.py`）

| 超参数 | 值 | 说明 |
|--------|-----|------|
| 学习率 | 2e-5 | BERT 微调常用的小学习率 |
| Batch Size | 32 | 每批处理 32 条数据 |
| Epochs | 5 | 完整遍历数据集 5 次 |
| 优化器 | AdamW | 带权重衰减的 Adam |
| 评估指标 | Accuracy（准确率） | 验证集上选择最佳模型 |

### 6.3 NER 训练（`train_ner.py`）

| 超参数 | 值 | 说明 |
|--------|-----|------|
| 学习率 | 5e-5 | NER 需要稍大的学习率 |
| Batch Size | 16 | 比分类小（NER 序列更长） |
| Epochs | 20 | NER 数据少，需要更多轮次 |
| 类别权重 | Inverse Frequency | 处理 O 标签占多数的类别不平衡 |
| 评估指标 | Macro F1 | 考虑实体级别的匹配 |

**NER 训练的特殊处理**：
- 计算类别权重，因为 O 标签占据了绝大多数（非实体部分），需要给实体标签更高权重
- 权重公式：`weight = total_count / (num_tags × tag_count)`

### 6.4 训练结果

| 模型 | 指标 | 成绩 |
|------|------|------|
| 文本分类（500条测试） | Accuracy | **100%** |
| 文本分类（~20000条测试） | Accuracy | **93.29%** |
| NER - PER（人名） | F1 | **0.964** |
| NER - LOC（地名） | F1 | **0.933** |
| NER - ORG（机构） | F1 | **0.911** |
| NER - Macro F1 | F1 | **0.936** |

> **一句话总结**：训练过程由通用训练器控制，分类用 5 轮 + 低学习率，NER 用 20 轮 + 类别权重，最终两个任务都达到了优秀水平。

## 九、技术难点与解决方案

### 难点 1：Encoder 权重冲突

**问题**：训练时分类和 NER 共用一个 encoder，但训练是分开的，各自保存了不同的 encoder 权重。推理时加载分类权重会覆盖 NER 权重。

**解决**：采用**双 Encoder 架构**（`DualEncoderModel`），两个 encoder 各自加载，互不影响。

### 难点 2：BIO 标签与 BERT 分词的对齐

**问题**：中文 BERT 使用字级别的 tokenizer，可能把一个词拆成多个 token。例如"姚明"可能被拆成"姚"和"明"。

**解决**：在 `NERDataset` 中，每个词的第一个子词保留原标签，后续子词继承相同标签。

### 难点 3：NER 类别不平衡

**问题**：BIO 标注中 O 标签占大多数（约 85%），模型容易偏向预测 O。

**解决**：使用 inverse frequency 类别权重，给实体标签（B-PER, I-PER 等）更高的损失权重。

### 难点 4：HuggingFace 网络访问问题

**问题**：中国大陆无法直接访问 HuggingFace，导致 tokenizer 下载失败，出现 `[UNK]` 问题。

**解决**：使用 `HF_ENDPOINT=https://hf-mirror.com` 从镜像站下载，并缓存到本地 `bert_tokenizer_cache/`。

### 难点 5：CRF 与 CE 的兼容性

**问题**：代码默认使用 CRF，但实际训练用了 CE，导致加载模型时出现参数不匹配错误。

**解决**：在加载 NER 模型时显式设置 `use_crf=False`，并用 `strict=False` 忽略 buffer 差异。

## 十、老师可能提问的问题与答案

### Q1：你们为什么选择 BERT 而不是其他模型（如 LSTM、GPT）？

**答**：BERT 是双向 Transformer 模型，能同时看到上下文前后的信息，这对理解中文语义非常重要。LSTM 是单向的，GPT 也是单向（自回归）的，在"理解"任务上不如 BERT。另外 bert-base-chinese 是专门的预训练中文模型，有良好的中文基础。

### Q2：什么是"预训练 + 微调"范式？

**答**：预训练是指在大规模语料上先训练一个通用模型（Google 帮我们做了），让模型学会基本的语言知识。微调是指在这个通用模型的基础上，用我们的具体任务数据做少量训练，让模型适应我们的任务。优势是即使数据少（如 NER 只有几百条），也能取得好效果。

### Q3：分类和 NER 有什么区别？

**答**：
- **分类**：给整段文本打一个标签（如"体育"），是"句子级别"的任务
- **NER**：找出文本中每个有意义的实体，是"词级别"的任务，需要标注每个字的标签

### Q4：为什么 NER 用 Macro F1 而不是 Accuracy？

**答**：NER 任务中 O 标签占绝大多数（约 85%），如果全预测 O，Accuracy 也能有 85% 左右，但这没有意义。F1 兼顾了 Precision 和 Recall，Macro F1 对每种实体类型平等对待，更能反映模型对实体识别的真实能力。

### Q5：什么是 BIO 标注？为什么需要它？

**答**：BIO 是一种序列标注方案。B(Begin) 表示实体开始，I(Inside) 表示实体内部，O(Outside) 表示非实体。需要 B 和 I 的区分是因为相邻的同类实体会被误认为一个（如"张三和李四"），B 标记可以区分边界。

### Q6：你们的模型分别用了什么损失函数？

**答**：
- 分类：CrossEntropyLoss（交叉熵损失），标准的多分类损失
- NER：CrossEntropyLoss + 类别权重，因为训练时没用 CRF，而是用 CE 配合逆频率权重解决类别不平衡

### Q7：什么是 CRF？为什么你们训练没用但代码里保留了？

**答**：CRF（条件随机场）能对标签之间的转移关系建模。例如 B-PER 后面只能接 I-PER 或 O，不能直接接 B-LOC。但本项目实际训练用的 CE + 类别权重也取得了 0.936 的 F1。代码保留 CRF 是为了灵活性和将来可能的对比实验。

### Q8：双 Encoder 架构比共享 Encoder 有什么优势？

**答**：共享 Encoder 需要两个任务一起训练（多任务学习），但本项目分类和 NER 是分开训练的，各自有最优权重。如果共享一个 encoder，后加载的模型会覆盖先加载的。双 Encoder 保证两个任务都用各自训练出的最佳权重，互不干扰。

### Q9：你们的数据集是怎么划分的？

**答**：按 80% 训练集 / 10% 验证集 / 10% 测试集的标准划分。训练集用于更新模型参数，验证集用于调参和选择最佳模型，测试集只用于最终评估，不参与训练。

### Q10：什么是梯度累积？为什么需要它？

**答**：梯度累积是指多次前向传播后才更新一次参数。例如 batch_size=16，累积 2 步，等效于 batch_size=32，但显存占用只有一半。这在 GPU 显存不足时非常有用。

### Q11：你们为什么设置不同的学习率？分类 2e-5，NER 5e-5？

**答**：NER 的数据量比分类少很多（几百条 vs 20 万条），需要更大的学习率来加快收敛。分类数据充足，小学习率能让训练更稳定。另外 NER 需要微调更深层的 BERT 参数，稍大的学习率有助于序列标注任务的学习。

### Q12：如果输入 NBA 这样的英文缩写会怎样？

**答**：`bert-base-chinese` 的词表中不包含英文缩写，NBA 会被 tokenize 成 `[UNK]`（未知 token）。这是该模型的一个已知限制。虽然分类任务能从上下文中推断出"体育"，但 NER 对英文实体的识别会受影响。如果需要更好处理混合文本，可以选择支持中英混合的模型。

### Q13：项目中有哪些创新点或亮点？

**答**：
1. **双 Encoder 架构**：解决了多任务独立训练后的推理权重冲突问题
2. **类别权重设计**：使用逆频率权重处理 NER 中的标签不平衡
3. **自定义前端**：纯 HTML/CSS/JS，零外部依赖，莫兰迪配色，响应式设计
4. **统一的 CLI 入口**：一条命令完成训练/推理/部署
5. **FastAPI + 自动化 API 文档**：Swagger 文档自动生成
6. **完善的错误处理**：CRF/CE 兼容、边界检查、网络镜像等

### Q14：你觉得这个项目还有哪些可以改进的地方？

**答**：
1. 可以尝试多任务联合训练（同时优化分类和 NER 损失），共享 encoder
2. 可以尝试更大的中文模型（如 RoBERTa、ERNIE 3.0）
3. NER 可以增加更多实体类型（时间、数量等）
4. 可以加入数据增强提高小样本类别（如时尚）的效果
5. 前端可以增加批量分析和历史记录功能
6. 可以封装为 Docker 镜像方便部署

### Q15：项目的整体流程是怎么样的？
