## 二、遇到的问题及修复方案

### 问题 1: `RuntimeError: Missing key(s) in state_dict: "crf.start_transitions"...`

**现象：**
```
RuntimeError: Error(s) in loading state_dict for NERTagger:
    Missing key(s) in state_dict: "crf.start_transitions", "crf.end_transitions", "crf.transitions".
    Unexpected key(s) in state_dict: "ce_loss.weight".
```

**原因：**
- `NERTagger.__init__` 默认参数 `use_crf=True`，创建了 CRF 层
- 但检查点 `best_ner.pt` 是按 `use_crf=False` 训练的，包含的是 `ce_loss.weight` 而非 CRF 参数
- `demo.py` 和 `main.py` 加载模型时未指定 `use_crf=False`

**涉及文件：**
- `src/app/demo.py` 第 43 行
- `main.py` 第 67 行
- `src/models/multitask_model.py` 第 96 行（硬编码 `.crf.decode()`）

**修复方案：**
1. 加载 NER 模型时显式设置 `use_crf=False`
2. 使用 `strict=False` 忽略 `ce_loss.weight` 的 buffer 差异
3. `MultiTaskModel.predict` 中根据 `self.ner_head.use_crf` 选择解码路径

```python
# 修复前（demo.py / main.py）
ner_head = NERTagger(encoder, num_tags=config["ner"]["num_tags"])
ner_head.load_state_dict(torch.load(ner_path, map_location="cpu"))

# 修复后
ner_head = NERTagger(encoder, num_tags=config["ner"]["num_tags"], use_crf=False)
ner_head.load_state_dict(torch.load(ner_path, map_location="cpu"), strict=False)
```

```python
# 修复前（multitask_model.py predict）
predictions = self.ner_head.crf.decode(emissions, mask=mask)[0]

# 修复后
if self.ner_head.use_crf:
    predictions = self.ner_head.crf.decode(emissions, mask=mask)[0]
else:
    pred_ids = torch.argmax(emissions, dim=-1)[0]
    predictions = pred_ids[:valid_len].tolist()
```

### 问题 2: Tokenizer 下载超时 → `[UNK]` 泛滥

**现象：**
- 推理时所有中文 token 都变成 `[UNK]`（未知）
- HuggingFace 连接超时：`WinError 10060`

**原因：**
- HuggingFace 在中国大陆网络环境不可直连
- BERT tokenizer 文件（`vocab.txt` 等）未缓存到本地
- 之前的训练可能在不同网络环境下完成，tokenizer 文件未被持久化

**涉及文件：**
- `src/data/dataset.py` 第 120 行
- `src/models/multitask_model.py` 第 36 行
- `src/models/encoder.py` 第 20 行

**修复方案：**
1. 使用 `HF_ENDPOINT=https://hf-mirror.com` 环境变量从镜像站下载
2. 所有 `from_pretrained()` 调用保持 `local_files_only=True` 确保后续离线可用

```bash
# 下载 tokenizer（仅需执行一次）
HF_ENDPOINT=https://hf-mirror.com python -c "
from transformers import BertTokenizer
tokenizer = BertTokenizer.from_pretrained('bert-base-chinese')
tokenizer.save_pretrained('bert_tokenizer_cache')
"
```

### 问题 3: `_parse_bio` 标签越界

**现象：**
- 当预测标签 ID 超出 `id2tag` 范围时可能崩溃

**原因：**
- 原代码 `tag_id < len(self.id2tag)` 未检查负值

**修复方案：**
```python
# 修复前
tag = self.id2tag[tag_id] if tag_id < len(self.id2tag) else "O"

# 修复后
tag = self.id2tag[tag_id] if 0 <= tag_id < len(self.id2tag) else "O"
```

### 问题 4: Gradio 端口占用

**现象：**
```
OSError: Cannot find empty port in range: 7860-7860.
```

**原因：**
- 前一个 Gradio 进程未正常退出，端口 7860 被占用

**修复方案：**
```bash
# 查找并终止占用端口的进程
netstat -ano | grep 7860
taskkill //F //PID <PID>
```

## 四、预防措施建议

1. **环境一致性**：所有下载操作统一设置 `HF_ENDPOINT=https://hf-mirror.com`
2. **检查点兼容性**：保存时记录 `use_crf` 等关键配置到 metadata，加载时自动匹配
3. **CI/CD**：在 `requirements.txt` 中固定依赖版本，避免 Gradio 等框架的 breaking change
4. **端口管理**：Gradio demo 使用 `find_available_port` 而非固定 7860
